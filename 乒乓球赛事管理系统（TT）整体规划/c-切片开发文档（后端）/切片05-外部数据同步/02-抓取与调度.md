# 02 - 抓取与调度（S14）

> **技术栈**：见 [`a-锚定项/02-技术栈.md`](../../../a-锚定项/02-技术栈.md)（Java 17 · Spring Boot · RabbitMQ 可选）

## 实现要点

- **解析器插件化**：`parser_type` 如 `html_v1`；实现类 `HtmlV1Parser`（**Jsoup** 解析 HTML）。
- **同步服务**：`ExtSyncService.runSyncSource(sourceId)` — `RestTemplate` 拉取 → 解析 → 写入 `ext_feed_item`，`(source_id, external_url)` 去重；失败写 `ext_sync_log`。
- **触发方式**：
  - 管理端 `POST /api/admin/ext-sources/{id}/sync`
  - `tt.sync.mode=direct`（默认）：请求线程内同步执行
  - `tt.sync.mode=queue`：消息入队 **`tt.ext.sync`**，由 `ExtSyncListener` 消费（需 RabbitMQ，见 Docker Compose）
- **定时**：`ExtSyncScheduler` 每分钟扫描启用的数据源，距上次同步 ≥ `interval_minutes` 则自动触发（`tt.sync.scheduler.*` 可配置）；卡住超过 30 分钟的 `running` 日志会自动标记失败。

## 代码落点

| 组件 | 路径 |
|------|------|
| 同步逻辑 | `tt-application/.../ext/ExtSyncService.java` |
| 解析器 | `tt-application/.../ext/HtmlV1Parser.java` |
| MQ 消费 | `tt-application/.../mq/ExtSyncListener.java` |
| 定时调度 | `tt-application/.../ext/ExtSyncScheduler.java` |
| 分发 | `tt-application/.../ext/ExtSyncDispatcher.java` |
| API | `AdminController` `/ext-sources/{id}/sync` |

## 验收

手动触发后 `ext_feed_item` 有新行且去重生效；失败时 `ext_sync_log.status=error` 且含 `error_message`。
