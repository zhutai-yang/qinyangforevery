# 02 - 数据库迁移与种子（S02）

## 最小表集

`fnd_user`、`fnd_role`、`fnd_user_role`；`evt_event`、`evt_venue`、`evt_event_venue`；`sch_stage`、`sch_match`、`sch_match_participant`；`reg_athlete`；`rec_result`。

## 索引建议

- `sch_match(event_id, scheduled_at)`
- `sch_match_participant(athlete_id)`

## 脚本与种子（IMP 治理链）

| 步骤 | 路径 |
|------|------|
| 权威 DDL | `tt-admin-api/src/main/resources/db/schema.sql` |
| 初始数据 | `db/data.sql` |
| 增量迁移 | `db/migration/V*.sql`（Flyway） |
| 本地一键 | `tt-admin-api/scripts/init-db.sh` |
| Docker 初始化 | `docker/mysql/init/01-schema.sql` + `02-data.sql` |

启动时 **DataInitConfig** 写入管理员（`ADMIN_INITIAL_PASSWORD` / 默认 `admin123`）；静态种子（角色、字典、首页块）已在 `data.sql`。
